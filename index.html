📖 О языке CRS

CRS (Cursed Coding Style) — это простой язык для создания веб-страниц. Он превращает человеко-понятные теги в настоящий HTML, CSS и JavaScript.

🔤 Синтаксис:
Каждая команда открывается тегом <c.имя_команды> и закрывается <cc.имя_команды>. Внутрь можно вкладывать другие команды.
⚙️ Как это работает:
1. Весь код пишется внутри <CURSED-CODING> ... </CCURSED-CODING>
2. Парсер превращает теги в реальные HTML-элементы
3. CSS и JS можно добавить через <c.style> и <c.script>

⚡ Правила синтаксиса
  • 1. Обязательная обёртка — весь код должен быть внутри <CURSED-CODING> ... </CCURSED-CODING>
  • 2. Закрывающие теги — строго <cc.имя_команды> (не </c.имя>)
  • 3. Регистр — все команды пишутся нижним регистром (c.button, c.text...)
  • 4. Вложенность — команды можно вкладывать друг в друга (например, c.box содержит c.text)
  • 5. Спецсимволы — внутри c.code используйте &lt; вместо < для демонстрации кода
  • 6. Ссылки и картинки — используют разделитель | (вертикальная черта): URL | текст
  • 7. Ошибки — при синтаксической ошибке парсер укажет строку и причину
💡 Совет: Для отладки используйте c.script с console.log() — сообщения появятся в консоли браузера.
❓ Часто задаваемые вопросы
❔ Почему кнопка не нажимается?
В CRS кнопки автоматически получают alert с текстом. Если внутри пусто — может не сработать. Используйте <c.script> для сложной логики.
❔ Как показать пример кода внутри страницы?
Используйте команду <c.code> и внутри неё экранируйте угловые скобки: &lt;c.button&gt;
❔ Можно ли использовать свои CSS-классы?
Да, через <c.style> вы можете задать любые стили. Также можно вставить готовый CSS фреймворк.
❔ Поддерживаются ли внешние библиотеки?
Да, через <c.script> можно написать любой JavaScript или подключить внешние скрипты через c.html.
❔ Что делать, если вижу "Неизвестная команда"?
Проверьте: закрывающий тег <cc.команда> должен точно совпадать по имени с открывающим. И все маленькими буквами.
